# Chapter 12: Staying Safe in Crypto

## Core Idea
Crypto has little forgiveness and little insurance. Security must be treated as a daily habit that limits damage from phishing, malicious contracts, compromised accounts, bad custody, and social engineering.

## Frameworks Introduced
- **Custody layering**: split funds by purpose and risk.
  - When to use: before holding meaningful amounts.
  - How: keep trading funds hot, long-term funds cold, browsing wallets separate, and large holdings diversified or multisig.
- **URL and source verification**: links are attack surfaces.
  - When to use: before connecting a wallet or downloading anything.
  - How: use official channels, bookmarks, mutuals, comments, and exact URL checks; distrust ads and unsolicited links.
- **Seed phrase survivability**: backups must resist theft, fire, water, memory loss, and panic.
  - When to use: when creating any wallet.
  - How: write on paper or metal, store copies separately, avoid cloud/photos/notes, and test recovery.
- **Social engineering skepticism**: unsolicited friendliness can be a long scam.
  - When to use: in DMs, community groups, job offers, "alpha" sharing, bot testing, or support messages.
  - How: slow down, ask a trusted person, verify independently, and never share seed/private keys.

## Key Concepts
- **CEX**: centralized exchange; convenient but not self-custody.
- **Hot wallet**: internet-connected wallet used for active trading and dapps.
- **Cold wallet**: hardware/offline key storage for significant holdings.
- **Seed phrase**: 12-24 word master recovery key.
- **Multisig**: wallet requiring multiple approvals.
- **Phishing**: fake link/site designed to steal funds or signatures.
- **Fake support**: scammer impersonating help staff.
- **Hacked account**: trusted social account used by an attacker.
- **Fake airdrop**: token or link bait that triggers malicious interaction.

## Mental Models
- Use **only the paranoid survive** as the default security posture.
- Think **blast radius**: if one wallet is drained, what survives?
- Treat **every wallet signature as a transaction with consequences**.
- Use **privacy as security**: portfolio size should not be public gossip.

## Anti-patterns
- **Keeping all funds on exchanges**: freezes, hacks, and bankruptcies can trap assets.
- **Storing seed phrases digitally**: notes apps, cloud drives, and screenshots are common leak points.
- **Connecting main wallets to unknown sites**: one bad approval can drain funds.
- **Trusting checkmarks or familiar accounts alone**: verification badges and hacked accounts are not safety.
- **Interacting with mystery tokens**: unknown airdrops can be malicious.

## Worked Example
A community post claims the dev is live and links to a launchpad lookalike. Before connecting, inspect the URL character by character, verify the link from the project's official profile and other channels, read comments for organic validation, and use a low-value browsing wallet if interaction is truly necessary. If anything feels rushed or too good, do nothing.

## Key Takeaways
1. Keep only necessary funds in hot wallets and exchanges.
2. Use cold wallets for meaningful long-term holdings.
3. Store seed phrases offline, redundantly, and carefully.
4. Verify URLs and sources before connecting wallets.
5. Be deeply cautious with DMs, downloads, job offers, fake alpha, and support requests.
