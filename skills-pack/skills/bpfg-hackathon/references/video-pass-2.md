## 2. The Village Loop (L1–L3 sensing → T1–T3 transmit)

From Village Builder Handbook slide:

### L1 — Sensing
- Scouts go into territory: shadow sanctuaries, digital hearths, spaces where people speak **before anyone important is watching**
- **Fights: Bias** — can’t rely only on what the model already holds
- Tools: Scout network (Reddit, LinkedIn, TikTok, Substack, podcasts…)

### L2 — Synthesizing
- Multiple perspectives held simultaneously, structured to disagree
- Elder Council processes what scouts bring back
- **Fights: Sycophancy + Dependency**
- Elders: different philosophies, blind spots, deliberate tension
- Friction produces truth; keeps **you** as intelligence in the room
- Tools: Elder Council — *challenge, not confirm*

### L3 — Shaping
- Focused questions that direct where to go in latent space
- Testing, prototyping, segment channeling
- **Fights: Drift** — without directed questions, model defaults to average
- Brief Builder, Segment Channeler = tools that stop gravitational pull to bland center

### T1 — Strategizing
- Definitive naming: what was heard becomes decisions, positioning, communication **in your voice, not the model’s**
- Builds **sovereignty**
- Insights Storytelling, voice-keeping skills

### T2 — Structuring
- Methodology made transmissible: villages, frameworks, skills → intelligence portable
- Builds **portability** — skills travel into client rooms, collabs, next engagement
- Village Archivist, Commons Builder, Skill Architect
- Skill files = your IP, machine-readable

### T3 — Syncing
- What goes out comes back as signal
- Every transmission feeds next round of listening
- Builds **continuity** — loop compounds, not repeats
- Archive is memory that makes this possible
- Archive + Commons + next brief

**Success definition (Day 2):**  
If you have **scouts + elders (sensors + shapers = L1–L2 + basic L3)** you are already very good.  
T2–T3 Commons/automation is **later** — not required for first real work.

## 3. Three stacked diversity layers (anti-bias)

1. **Human–model 1:1** can create bias/sycophancy fast  
2. **Territory diversity** in briefs (different places)  
3. **Model diversity** (Claude / Gemini / Grok / ChatGPT / Perplexity council) — each founder’s philosophy shapes answers  

Perplexity Max: **model council** = one question → multiple models → similarities/differences.

ChatGPT more sycophantic / thesis-confirming if brief is leading.  
Claude: safety-heavy, sometimes over-interprets, sometimes holier-than-thou.  
Always challenge sources when synthesis overreaches.

## 4. Trust / validation stack

### Participant trust frame (Daily)
Data for decisions should be:
1. Transparent  
2. Rigorous  
3. Unbiased  
4. Segmented  
5. **Triangulated**

### Structural Truth (Abi’s personal system)
- Separate skill from exploration (~30 tests on an insight)
- Sources, numbers, bias, language distortion, harm checks
- Too heavy to ship full; **simplified validation skill** is enough for most
- Logic: **always separate explore skills from stress-test skills**

### Quantification angles (when needed)
- Volume, repetitiveness, triangulation, longevity (when issue first appeared / how long unserved)
- Optional brief gate: “only if N people…” — possible but **against pure leading-edge intent**
- Prefer intensity + multi-place echo first; size later with survey/internal data

### Bot hygiene
- Web is bot-polluted; pure automation of insight → slop risk  
- Can build bot-pattern rejection skills

## 5. Brief anatomy (full commission fields)

From live brief walkthrough:
- Overall / real question  
- **Living question** (non-analytical; opens insight)  
- Where to look (systematic places)  
- Language/patterns to listen for (not only keywords)  
- Shadow / hard emotional material  
- **Filter out** list (noise genres)  
- Desired outputs  
- **Deploy options** (Perplexity deep research, Grok, ChatGPT research, etc.)  

**Separate briefs per territory** usually better than one mega-brief; deploy to different agents; unify later.

**Always start with Brief Builder** for new research (confirmed Q&A).

**Ongoing practice once village lives:**  
Voice intent → Brief Builder → Scouts → Elders → (optional) visualize / validate / package / decide

## 6. Platform / skill anatomy (how good scouts are built)

### Instagram (and similar visual) skill pattern
1. Educate agent on **what the platform is for** (ontology)  
2. Modes: URL, grid, hashtag, discovery, search  
3. Tools: JS, screenshots, APIs, transcript services  
4. **Embed analytical frameworks** (semiotics, identity, cultural analysis)  

Same pattern: Pinterest for packaging/semiotics; TikTok for emotion/technique.

### TikTok two paths
| Path | What it gets |
|---|---|
| **TokScript MCP** (~£10; free ~5/day no MCP) | Transcripts, bulk, images; Claude + GPT |
| **TikTok Chrome Scout** | Behavior, scenes, occasions, background products, face/emotion context |

Can convert recurring browser plans → **scheduled tasks**.

### Access rules
- Tick marks = built-in scout available  
- Facebook groups / LinkedIn paywall / Instagram login → **browser while logged in**  
- Ethics: skills may refuse personal-info tasks — correct  
- Read ≠ scrape; platforms block scrapers more than human-like reading

## 7. Model & tool discipline

- **Mental real estate:** use **2–3 tools**, not FOMO all tools  
- Models = bigger telescopes; care more about **new tools/connectors** than raw size  
- Sonnet-class for speed; Opus for long advanced prompts (credit burn)  
- Haiku sometimes best for mechanical speed (live note)  
- Claude Pro can exhaust on video-heavy; Max more resilient  
- Platform-agnostic skills; Claude recommended for native skills + Co-work  
- Portable archives: if Claude dies / KYC / local AI — you leave with Drive/Obsidian  

### Village shells are any UI
Claude, Perplexity, Gemini, NotebookLM, Manus, Grok, ChatGPT — all can be “villages” if configured. Claude highlighted for skills.

## 9. Elder / scout operational details

- System prompt stubs are **flimsy**; ops manuals in knowledge files = depth  
- Blind spots + tension pairs are non-negotiable  
- Triggers: manual `@Name` OR proactive (“when I circle / unsure / mention X”)  
- Option A: full council · Option B: lead pair (e.g. Columbo + Decider)  
- ~**5 scouts enough**; 11 becomes painful under one-at-a-time builder  
- New chat per scout preferred as **change-log home** for that skill  
- “Please make the skill” → download → upload global or project  
- Update path: new chat + HMG principles — don’t blindly duplicate  
- **Village audits** after structural changes (redundancy, broken @deps)  
- Skills that @mention Brief Builder / other scouts — audit dependencies  
- Prefer **regenerate full system prompt** over fragile snippet splice  
- Trial: if elder too invasive, tighten grip  
- Skills guide process — ask “what do I do next?” when lost  
- Over-friction in Scout Builder intentional to slow users; Co-work/Code for batch  

### Scout blueprint types (examples from live)
- Blind-spot detective  
- Fringe / Grace Jones antenna  
- Human truth excavator  
- Cultural moment mapper  
- Client whisperer  
- Budget/ambition check  

Elders ≠ scouts (ethnographer often **elder**).

## 15. Anti-patterns (extra from pass 2)

- Entering at Layer 4–5 only (surveys/clicks) and missing Layer 1  
- Treating L1 sensing as complete without L2 council friction  
- Skipping L3 shaping → drift to average  
- Shipping model voice as your strategy (failed T1 sovereignty)  
- 11 scouts day one  
- Skills only global when confidential  
- No audit after edits  
- Skill Creator vs Architect confusion  
- Relying on untested quant scout for pitch claims  
- Ignoring bot-filled web when automating  
- One mega-brief for all territories when separate is sharper  
- FOMO tool stack burning mental real estate
